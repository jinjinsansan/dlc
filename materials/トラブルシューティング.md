## インストール・環境構築

### Q1. `npm install -g @anthropic-ai/claude-code` で `permission denied`

**原因**: 管理者権限が必要

**解決法**:
- **Mac**: コマンドの先頭に `sudo` を付ける（パスワード求められる）
  ```bash
  sudo npm install -g @anthropic-ai/claude-code
  ```
- **Windows**: PowerShell を **「管理者として実行」** で開き直す

### Q2. `node: command not found` （Node.js が入っていない）

**解決法**:
- **Mac**: `brew install node`
- **Windows**: [https://nodejs.org](https://nodejs.org) から LTS をダウンロード&インストール

### Q3. `brew: command not found` （Homebrew が入っていない、Mac）

**解決法**:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
インストール後にターミナルを再起動して再試行。

### Q4. インストールが途中で止まる・遅い

**原因**: ネットワークの問題

**解決法**:
- Wi-Fi を再起動
- 別の Wi-Fi（携帯回線等）で試す
- VPN を OFF にする

### Q5. ターミナルで日本語が文字化けする

**解決法**:
- **Mac**: 通常は問題なし。出る場合はターミナル設定で「テキストエンコーディング: UTF-8」を確認
- **Windows**: PowerShell を使う（コマンドプロンプトは日本語対応が弱い）

## Claude Code 起動・操作

### Q6. `claude: command not found`

**原因**: インストール直後はターミナルが古い情報

**解決法**:
1. ターミナルを完全に閉じる
2. 新しく開く
3. もう一度 `claude` を実行

それでも出ない場合、再インストール:
```bash
npm install -g @anthropic-ai/claude-code
```

### Q7. ログイン画面が開かない

**解決法**:
ターミナルに表示される URL を、ブラウザのアドレスバーに **手動でコピペ** してください。
通常は自動でブラウザが開きますが、まれに開かない環境があります。

### Q8. 何度も「許可しますか?」と聞かれて煩わしい

**解決法**:
プロンプトで `a` を入力すると **「セッション中はすべて許可」** になります。
慣れた人はこれで OK。

### Q9. Claude が同じことを繰り返してループする

**解決法**:
1. `Ctrl + C` を 1 回押して中断
2. 「ストップ。一度落ち着こう。○○について再度説明して」と頼む
3. 別の頼み方に変える

### Q10. Claude の応答が異常に遅い

**原因**:
- API の負荷
- 大きなファイルを処理中
- ネットワーク

**解決法**:
- 待つ（通常 1〜2 分で完了）
- 別タブでネットワーク確認
- 一度 `/exit` で終わって再起動

## Web 表示・デザイン

### Q11. ブラウザでサイトを開いたら真っ白

**原因と解決法**:
1. **JavaScript エラー** → 開発者ツール（F12）の Console タブを見る → エラー文を Claude に渡す
2. **ファイルパスが間違っている** → URL を確認、`index.html` を直接開いてみる
3. **HTML が壊れている** → 「画面が真っ白。HTML を確認して直して」

### Q12. 変更を加えたのに、ブラウザで反映されない

**原因**: ブラウザのキャッシュ

**解決法**:
- **Mac**: `⌘ + Shift + R`
- **Windows**: `Ctrl + Shift + R`

それでも反映されない場合、開発者ツールを開いて「Disable cache」にチェック。

### Q13. 画像が表示されない

**原因**:
- 画像ファイルのパスが間違い
- ファイル名の大文字小文字違い

**解決法**:
```
画像が表示されません。
images/profile.jpg に置いたのですが、表示されない。
パスとファイル名を確認して直して。
```

### Q14. スマホで見たら横スクロールができてしまう

**原因**: どこかの要素が画面幅を超えている

**解決法**:
```
スマホで横スクロールが発生しています。
原因の要素を特定して、はみ出さないよう修正して。
```

### Q15. フォントが効いていない

**原因**: フォントファイルの読み込みが完了していない or 指定が間違い

**解決法**:
```
Google Fonts の Noto Sans JP を読み込んで、全ての日本語に適用して
```

## Supabase・認証

### Q16. 「Invalid API key」と出る

**原因**: `.env.local` の Key の貼り付けミス

**解決法**:
1. `.env.local` を開く
2. Key の前後に余分なスペース・改行・ダブルクォートが無いか確認:
   ```
   ✅ NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...（クォートなし）
   ❌ NEXT_PUBLIC_SUPABASE_ANON_KEY="eyJhbGc..."（クォート不要）
   ```
3. 開発サーバを再起動（`Ctrl+C` → `npm run dev`）

### Q17. メール認証メールが届かない

**解決法**:
- **テスト中**: Supabase の「Authentication > Settings」で「Confirm email」を **OFF** にする
- **本番**: SMTP 設定を確認、または SendGrid 等の外部メールサービスに切り替え

### Q18. ログインしたのにすぐログアウトされる

**原因**: Cookie / セッションの設定問題

**解決法**:
```
ログインしてもすぐログアウトされてしまう。
Cookie の設定とセッション管理を確認して直して
```

### Q19. Supabase の RLS (Row Level Security) でデータが取れない

**原因**: RLS が有効化されているが、ポリシーが設定されていない

**解決法**:
```
○○テーブルの RLS ポリシーを設定して。
- 認証済みユーザーは自分のデータだけ読み書き可能
- 管理者（ADMIN_EMAIL）は全て可能
```

### Q20. テーブルを間違って作成・削除した

**解決法**:
- Supabase の「Table Editor」で削除 → SQL を作り直して再実行
- 開発初期はデータを失っても OK な前提で進める
- 本番後は **DB バックアップ機能** を有効化

## API キー・AI 機能

### Q21. AI チャットが「API key not found」と出る

**原因**:
- `.env.local` に `ANTHROPIC_API_KEY` が無い
- Vercel に環境変数を設定していない

**解決法**:
1. `.env.local` に `ANTHROPIC_API_KEY=sk-ant-xxx` を追加
2. Vercel の Settings → Environment Variables にも同じものを追加
3. 開発サーバを再起動

### Q22. AI の応答が遅い

**解決法**:
モデルを軽量なものに変える:
```
モデルを claude-haiku-4-5 に変えて。応答速度を優先する。
```

| モデル | 速度 | 賢さ | コスト |
|--------|------|------|--------|
| haiku | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | $ |
| sonnet | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | $$ |
| opus | ⭐⭐ | ⭐⭐⭐⭐⭐ | $$$$ |

### Q23. API 利用料が思ったより高い

**解決法**:
- haiku モデルに切り替える
- ユーザーごとに 1 日の利用回数制限を設ける
- 長文要約は文字数制限を設ける（5000 字までなど）
- Anthropic Console の「Usage」で消費を確認

## GitHub・Vercel・公開

### Q24. `git push` で「Permission denied」

**原因**: GitHub の認証が通っていない

**解決法**:
- GitHub Personal Access Token を作成して使う
- または GitHub CLI (`gh auth login`) でログイン

困ったら:
```
GitHub に push しようとしたら Permission denied と出た。手順を教えて
```

### Q25. Vercel デプロイがビルドエラーで失敗

**解決法**:
1. Vercel の画面でエラーログをコピー
2. Claude に貼り付け:
   ```
   Vercel ビルドで以下のエラー。直して:
   [エラー文]
   ```

頻出原因:
- 環境変数の設定漏れ
- ローカルにしかないファイル参照
- TypeScript エラー

### Q26. 本番デプロイは成功したが、サイトが真っ白

**原因**: クライアント側のエラー

**解決法**:
- 公開された URL で開発者ツール（F12）を開く
- Console タブのエラーをコピー → Claude に渡す

### Q27. 独自ドメインが繋がらない

**原因**: DNS 伝播待ち（最大 48 時間、通常 30 分〜2 時間）

**解決法**:
- Vercel の Domains 画面で「Valid Configuration」を待つ
- DNS 設定が間違っていないか Claude に確認:
  ```
  独自ドメイン yamada.com を Vercel に繋ぎたいが繋がらない。
  お名前.com で何を設定すればいい?
  ```

## Stripe・決済

### Q28. Stripe Webhook が動かない（決済しても DB が更新されない）

**原因**: Webhook signing secret 未設定 or 間違い

**解決法**:
1. Stripe Dashboard → Webhooks → 該当エンドポイント
2. **Signing secret** をコピー
3. `.env.local` と Vercel の両方に `STRIPE_WEBHOOK_SECRET=whsec_xxx` を設定

### Q29. テストカードでも「決済が拒否されました」

**解決法**:
- カード番号: **4242 4242 4242 4242** （半角・スペースなし）
- 有効期限: **任意の未来の日付**（12/35 等）
- CVC: **任意の 3 桁**（123 等）

それでも失敗するなら、Stripe ダッシュボードで「テストモード」になっているか確認。

### Q30. 本番モードに切り替えたら動かない

**原因**: 本番用 API キーへの切り替え漏れ

**解決法**:
- `.env.local` の API キーを `pk_live_xxx` / `sk_live_xxx` に変更
- Vercel の環境変数も全て本番用に変更
- 商品・プランも本番モードで再作成
- Webhook も本番用 URL で再設定
