# Week 5: AI の力をアプリに入れる

> **このWeekのゴール**: 自分のアプリに「AI 機能」を搭載する。他にない、自分だけの武器を持つ
> **所要時間**: 動画 50 分 + 教材 90 分 + 宿題 30 分
> **このWeekで身につく力**: 「AI を組み込んだサービス」を作る力。ここから本物の差別化が始まる

---

## このWeek を始める前に

### Week 4 までの状態

- ✅ ログイン・データ保存が動く Web アプリ
- ✅ デザインも整っている
- ✅ ほぼ「サービス」として完成形に近い

ただし、これだけだと **「ただの Web アプリ」** です。
今や Web アプリは作れる人が増えてきました。

### 「AI 搭載」が差別化の最大の武器

**今、最も求められているのが「AI を上手に使ったサービス」** です。

```
普通のメモ帳 → 100 万個ある
AI が文章を要約してくれるメモ帳 → ほぼ無い → 売れる

普通の家計簿 → 100 万個ある
AI が支出をアドバイスしてくれる家計簿 → ほぼ無い → 売れる

普通のブログ → 100 万個ある
AI が読者の質問に答えてくれるブログ → ほぼ無い → 集客できる
```

**「AI 機能を入れる」** を 1 行覚えるだけで、あなたのサービスは **「世にあるサービスの 1 つ」** から **「他にない武器を持つサービス」** に変わります。

## レッスン 5-1: アプリに入れられる AI 機能（10 分）

### あなたのアプリに入れられる AI 機能 7 選

#### ① AI チャット
```
ユーザーが質問 → AI が答える
例: カスタマーサポート Bot、相談相手アプリ
```

#### ② テキスト要約
```
長文を貼り付け → AI が 3 行に要約
例: ニュース要約、議事録要約、論文要約
```

#### ③ 文章生成
```
キーワードを入力 → AI が記事を書く
例: ブログ記事生成、SNS 投稿生成、商品説明生成
```

#### ④ 翻訳
```
日本語を入力 → 英語に翻訳（または逆）
例: 多言語ブログ、海外向け商品紹介
```

#### ⑤ 画像認識・分析
```
画像をアップ → AI が内容を説明
例: 食事の栄養分析、植物の名前判定、文書スキャン
```

#### ⑥ レコメンド
```
ユーザーの好み → AI が「おすすめ」を提案
例: ECの商品レコメンド、コンテンツレコメンド
```

#### ⑦ 自動分類・タグ付け
```
データを投入 → AI が自動でカテゴリ分け
例: メール自動分類、画像自動タグ付け
```

### あなたのアプリに何を入れる？

#### 自己紹介サイトなら...
- AI が自己紹介を読んで、訪問者の質問に答える「私の AI 分身」

#### サービス LP なら...
- AI が見込み客の質問に 24 時間答える Bot

#### ブログなら...
- AI が記事を要約・翻訳してくれる
- AI が「関連記事」を自動レコメンド

#### 趣味サイトなら...
- 競馬予想なら → AI が今週のレースを分析
- 料理なら → AI が冷蔵庫の食材から献立提案
- 読書なら → AI が「この本好きならこの本」をレコメンド

## レッスン 5-2: API キーの取得（コピペだけ）（10 分）

### Anthropic API キーを取得（推奨）

Claude を作っている Anthropic 社の API を使います。

#### ステップ
1. [https://console.anthropic.com](https://console.anthropic.com) にアクセス
2. アカウント作成（既に Claude Pro なら同じアカウントで OK）
3. 左メニューの「API Keys」をクリック
4. 「Create Key」をクリック → 名前を付ける（例: `my-website`）
5. **表示された API キーをコピー（一度しか表示されないので注意）**
6. メモ帳に保存

### 料金の目安
- 最初に **$5 のクレジット** を入金しておけば、Week 5 の練習で使い切ることはほぼ無いです
- 1 ヶ月の試行で $5〜$10 程度

### Claude に伝える

`.env.local` に以下を追加:

```
ANTHROPIC_API_KEY=sk-ant-xxxxxx...
```

そして Claude に頼む:
```
.env.local に ANTHROPIC_API_KEY を設定したので、
これを使って Claude API を呼べる仕組みを作って。

ライブラリは @anthropic-ai/sdk を使って。
```

→ Claude が必要なファイルを作ってくれます。

### OpenAI API も使えます

ChatGPT の API（OpenAI）もほぼ同じ手順:
1. [https://platform.openai.com](https://platform.openai.com)
2. API Keys から作成
3. `.env.local` に `OPENAI_API_KEY=sk-...` を追加

どちらを使うかは Claude に頼めば、適切な方を提案してくれます。
**おすすめは Anthropic（Claude API）**。日本語が綺麗、長文を扱える。

## レッスン 5-3: AI チャット機能をつける（20 分）

### 一番シンプルな実装

```
サイトに AI チャット機能をつけて。

【場所】
画面右下に固定で「💬 チャット」ボタン
クリックすると、チャットウィンドウが開く

【機能】
- ユーザーがメッセージを入力
- 送信ボタンを押すと AI が答える
- やり取りの履歴が見られる

【AI のキャラ設定】
親しみやすい執事のようなアシスタント
口調は「ですます調」で丁寧
このサイトのことについて何でも答えてくれる

【使う API】
Anthropic Claude API（モデルは claude-haiku-4-5）
```

→ Claude が API ルート + UI を一気に作ります。

### 動作確認

1. 右下に「💬 チャット」ボタンが出る
2. クリックでチャット欄が開く
3. メッセージを送信 → AI が答える ✨

これで **あなたのサイトに AI が住みました** 🎉

### 失敗しがちなポイント

- **API キーを `.env.local` に設定したか?** → 必須
- **API ルートはサーバー側で動くか?** → クライアントから直接 API を叩くと API キーが漏れる。Claude は自動でサーバー側に配置してくれます

## API キー取得手順書

### Anthropic Claude API の場合

1. https://console.anthropic.com/ を開く
2. ログイン or 新規登録
3. 左メニュー「API Keys」をクリック
4. 「Create Key」ボタン
5. キー名を入力（例: `my-website`）
6. **表示されたキーをコピー（一度しか表示されない）**
7. プロジェクトの `.env.local` に追加:
   ```
   ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxx
   ```
8. **Billing**:
   - 「Settings > Billing」で $5 入金
   - これで当面 OK

### OpenAI API の場合

1. https://platform.openai.com/ を開く
2. ログイン or 新規登録
3. 「API keys」をクリック
4. 「Create new secret key」
5. キーをコピー
6. `.env.local` に追加:
   ```
   OPENAI_API_KEY=sk-xxxxxxxxxxxxx
   ```
7. **Billing**: 「Settings > Billing」で $5 入金

### 重要な注意

- **API キーは絶対に GitHub に push しない**
- `.env.local` は `.gitignore` に入れる（Claude が自動で設定済みのはず）
- 万が一漏らしたら、ダッシュボードでキーを削除して再発行

## トラブルシューティング

### Q1. API キーを設定したのに「Invalid API key」と出る

**A**: チェックポイント:
- スペース・改行が混入していないか
- ダブルクォート (`"`) で囲んでいないか（`.env.local` ではクォート不要）
- 開発サーバを再起動したか（`Ctrl+C` で止めて `npm run dev`）

### Q2. AI の応答が遅い

**A**: モデルを変える:
```
モデルを claude-haiku-4-5 に変えて。応答速度を優先する。
```
- haiku（速い・安い）
- sonnet（バランス）
- opus（賢い・遅い・高い）

### Q3. AI が変なこと（嘘）を言う

**A**: システムプロンプトで縛る:
```
AI に対する指示を厳しくして。
- 知らないことは「分かりません」と答える
- 推測で答えない
- 「絶対」「必ず」などの断定表現を使わない
```

### Q4. API 料金が思ったより高い

**A**:
- haiku モデルに切り替える（10 倍以上安い）
- ユーザーごとに 1 日の利用回数制限を設ける
- 長文要約は文字数制限を設ける

### Q5. 「画像認識」を使いたいが、どのモデルが良い?

**A**: `claude-haiku-4-5` か `claude-sonnet-4-5` のマルチモーダル機能を使えば OK。Claude に頼めば対応コードを書いてくれます。

### Q6. AI チャットの履歴が消えてしまう

**A**:
```
チャット履歴を Supabase に保存して、
ユーザーがログインし直しても履歴が残るようにして
```
