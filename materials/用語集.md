> 受講中に分からない言葉が出てきたら、ここを開いてください。

## あ

### API（エーピーアイ）
**ざっくり**: アプリ同士が会話する仕組み
**例えるなら**: 飲食店の注文票（お客→キッチンへ）
**使う場面**: AI を組み込む（Week 5）、決済（Week 7）

### API キー
**ざっくり**: API を使うためのパスワード
**注意**: **絶対に GitHub に上げない・人に見せない**
**保管場所**: `.env.local` （ローカル）+ Vercel の Environment Variables（本番）

---

## い

### インポート（import）
**ざっくり**: 「他のファイルから○○を借りてくる」命令
**例**: `import Header from "@/components/Header"` = Header コンポーネントを借りる

---

## う

### Webhook（ウェブフック）
**ざっくり**: 「○○が起きたら、私のサーバーに教えて!」という仕組み
**例えるなら**: 宅配便の「お届け通知」
**使う場面**: Stripe の決済完了通知、GitHub の push 通知 等

---

## え

### 環境変数（envrionment variable）
**ざっくり**: コードに直接書きたくない秘密情報を、別ファイルに分離する仕組み
**保管場所**: `.env.local`
**例**: API キー、DB パスワード

---

## お

### OAuth（オーオース）
**ざっくり**: 「Google でログイン」「X でログイン」できる仕組み
**メリット**: ユーザーが新しいパスワードを覚えなくていい

### OGP 画像（オージーピー画像）
**ざっくり**: SNS で URL をシェアした時に表示されるサムネイル
**サイズ**: 1200x630px が標準
**設定場所**: HTML の `<meta property="og:image">` タグ

---

## か

### カスタムドメイン
**ざっくり**: 自分専用の URL（例: yamada.com）
**取得**: お名前.com / Cloudflare Registrar 等で年 1,000〜2,000 円
**Week 6** で扱う

### 環境（development / production）
**ざっくり**:
- **development**: 開発中のローカル PC
- **production**: 本番（公開された Vercel）

---

## く

### Cron（クーロン）
**ざっくり**: 「毎日朝 9 時に○○を実行」のような定期実行の仕組み
**使う場面**: 自動メール送信、自動 SNS 投稿、データ集計

### Claude Code
**ざっくり**: このアカデミーで使うメインツール
**正体**: 日本語で頼むと PC の中で本物のアプリを作ってくれる AI ツール

### CTA（シーティーエー）
**ざっくり**: Call To Action の略。「行動を促すボタン or リンク」
**例**: 「今すぐ申し込む」「無料で試す」「もっと見る」

---

## こ

### Console（コンソール）
**ざっくり**: ブラウザの「裏側」が見える画面
**開き方**: 右クリック → 「検証」→ Console タブ
**ここで**: エラー文を確認・コピー

### コンポーネント（component）
**ざっくり**: ボタン・カード・ヘッダーなど **「再利用できる部品」**
**例**: `Button` コンポーネントを 1 個作っておけば、どのページでも使える

### Cookie（クッキー）
**ざっくり**: ブラウザが覚えておく「あなたの情報」
**例**: ログイン状態、サイトの設定

### CV / Conversion（コンバージョン）
**ざっくり**: ユーザーに「やってほしい行動」を達成してもらうこと
**例**: 会員登録、購入、問い合わせ
**コンバージョン率（CV率）**: 訪問者のうち、何 % が達成したか

---

## さ

### サーバー（server）
**ざっくり**: 24 時間動き続けて、リクエストに応えるコンピュータ
**例**: Vercel が「Web サーバー」、Supabase が「DB サーバー」

### サブスクリプション（subscription）
**ざっくり**: 月額・年額の継続課金
**メリット**: 安定収入になる

### サンセリフ（sans-serif）
**ざっくり**: 「ヒゲのない」フォント（ゴシック体）
**例**: Helvetica、Noto Sans

---

## し

### CSS（シーエスエス）
**ざっくり**: 「見た目」を決める言語
**Claude Code 時代**: 自分で書く必要なし。「○○な見た目に」と頼むだけ

### CMS（シーエムエス）
**ざっくり**: ブログや記事を管理するシステム
**例**: WordPress、Notion

---

## せ

### セリフ（serif）
**ざっくり**: 「ヒゲのある」フォント（明朝体）
**例**: Noto Serif、Times New Roman
**印象**: 高級感・上品

### SaaS（サース）
**ざっくり**: Software as a Service の略
**意味**: 「ソフトウェアをサービスとして提供」= 月額課金で使うアプリ
**例**: Notion、Slack、Stripe

### SEO（エスイーオー）
**ざっくり**: Google 検索で上位に出るための工夫
**やること**: タイトル・description・コンテンツを整える

### Supabase（スーパーベース）
**ざっくり**: 個人開発者の親友。**DB + 認証 + ファイル保存** が無料で使えるサービス
**Week 4** で導入

### Stripe（ストライプ）
**ざっくり**: 世界最大の決済サービス
**手数料**: 売上の 3.6%
**Week 7** で導入

---

## た

### ターミナル
**ざっくり**: PC に文字でお願いする画面（黒い or 白い画面）
**Mac**: アプリケーション > ユーティリティ > ターミナル
**Windows**: PowerShell

---

## て

### デプロイ（deploy）
**ざっくり**: 「公開する」のかっこいい言い方
**例えるなら**: 出版社に原稿を送って本を流通に乗せる感じ

### テーブル（table）
**ざっくり**: データベースの「データの入れ物」
**例えるなら**: エクセルの 1 シート

---

## と

### ドメイン
**ざっくり**: あなた専用 URL（yamada.com 等）
**Week 6** で扱う

### TODO（トゥードゥー）
**ざっくり**: 「やることリスト」

---

## に

### Next.js（ネクストジェイエス）
**ざっくり**: Web サイト・Web アプリを作る人気フレームワーク
**Claude Code 時代**: 自分で深く理解しなくて OK。Claude が使ってくれる

---

## は

### ハンバーガーメニュー
**ざっくり**: スマホの右上にある三本線アイコン
**役割**: タップするとメニューが開く

### ハッシュタグ
**ざっくり**: SNS で `#` を付けた言葉
**役割**: 同じテーマの投稿が集まる

---

## ふ

### フレームワーク
**ざっくり**: アプリ開発の「土台」
**例**: Next.js（React の上に乗ったフレームワーク）

### フリーミアム
**ざっくり**: 「基本無料 + 有料プラン」のビジネスモデル
**例**: Notion（基本無料、チーム機能で有料）

### Pull Request / プルリクエスト（PR）
**ざっくり**: 「この変更を取り込んでください」のお願い
**GitHub** で使う

## ほ

**ホバーエフェクト**: マウスを乗せた時の見た目変化

### Webhook → 「ウェブフック」を参照

## ま

### マイグレーション（migration）
**ざっくり**: データベースの構造変更
**例**: テーブルに新しい列を追加

---

## ら

### ライブラリ（library）
**ざっくり**: 他の人が作った「便利なコード集」
**例**: React、tweepy、stripe-js

### ランディングページ（LP）
**ざっくり**: 「商品・サービスを売るための 1 ページ」
**特徴**: 縦にスクロールしていくと、最後に申込ボタンがある

---

## ろ

### ローカル（local）
**ざっくり**: 自分の PC の中
**↔ 本番（production）**: 公開されたサーバー

### ロゴ
**ざっくり**: ブランドの目印（画像・文字）

## A-Z

### CRUD（クラッド）
**ざっくり**: Create, Read, Update, Delete の略
**意味**: 「作る・読む・更新する・削除する」の 4 機能
**ほぼ全てのアプリ**で必要な基本機能

### DB（ディービー）
**= データベース**
**ざっくり**: データの保管庫
**例えるなら**: 巨大なエクセルファイル

### env / .env.local
**= 環境変数ファイル**
**ざっくり**: 秘密情報を入れるファイル
**注意**: GitHub に絶対に上げない

### JSON（ジェイソン）
**ざっくり**: データをやり取りする形式
**例**: `{"name": "山田", "age": 30}`

### Node.js（ノードジェイエス）
**ざっくり**: JavaScript を PC で動かすための土台
**Claude Code を動かすのに必須**

### npm
**ざっくり**: Node.js のライブラリを管理するツール
**Claude Code のインストールに使う**: `npm install -g @anthropic-ai/claude-code`

### React（リアクト）
**ざっくり**: Web アプリを作るライブラリ
**Claude Code 時代**: 知識不要。Claude が使ってくれる

### REST API
**ざっくり**: API の一般的な規格
**深く知る必要なし**: 「API の一種」と覚えておけば OK

### TypeScript
**ざっくり**: JavaScript の進化版（より厳格）
**Next.js 14 で標準**: Claude が自動で書いてくれる

### URL
**ざっくり**: Web ページの住所
**例**: https://yamada.com

### UX（ユーエックス）
**ざっくり**: User Experience = ユーザー体験
**「使い心地が良いか」** のこと

### UI（ユーアイ）
**ざっくり**: User Interface = 画面の見た目・操作性

### Vercel（バーセル）
**ざっくり**: 個人開発者の親友。**Web サイトを無料で公開できる**サービス
**Week 6** で使う

---

## おわりに

この用語集は **「全部覚える必要はない」** です。
分からない言葉が出てきたら、ここを開く。それだけで OK。

カタカナ語が多くて怖いかもしれませんが、本質は **どれも単純** です。
1 ヶ月もすれば、ほとんどの用語が「あ、それね」と分かるようになります。
