> **このWeekのゴール**: 決済機能が動くようになる。テスト決済で「受け取り」を完了する

## このWeek を始める前に

### 「お金を受け取る」ことの心理的な壁

多くの人がここで一番ためらいます。

> 「こんな自分が作ったものに、お金を払ってもらえるのだろうか」
> 「無料で使ってもらった方が良くない?」
> 「お金が絡むと面倒なんじゃ...」

**全部、ただの思い込み** です。

事実:
- AI 搭載の Web サービスは **「希少」** で、本当に価値があります
- 月額 980 円〜2,980 円なら、価値を感じれば誰でも払います
- 仕組みを作れば「お金を受け取る」は **本当に楽** です（Stripe のおかげで）

### 「お金の受け取り方」4 パターン

| 方式 | 特徴 | 向いている人 |
|------|------|------------|
| **一括払い** | 1 回払い切り。シンプル | 教材販売・買い切りツール |
| **月額課金（サブスク）** | 毎月自動課金。安定収入 | コンテンツサイト・継続利用ツール |
| **フリーミアム** | 基本無料・上位プランで課金 | 多くのユーザーを集めるサービス |
| **投げ銭・寄付** | 任意で支払い | クリエイター・ブロガー |

Week 7 では、まず **「一括払い」** と **「月額課金」** の両方を実装できるようにします。

## レッスン 7-1: お金の受け取り方 4 パターン（10 分）

### 詳細解説

#### ① 一括払い（買い切り）
```
ユーザーが「○○円を 1 回払う」→ 永久にサービス利用可能 or 商品ダウンロード

例:
- 教材販売（買い切り 9,800 円）
- ダウンロードソフト（5,000 円）
- アクセス権 6 ヶ月分（10,000 円）
```

メリット: シンプル・心理的ハードル低い
デメリット: 一回切りなので、継続的な収益にならない

#### ② 月額課金（サブスクリプション）
```
ユーザーが「月 980 円」を毎月自動で払う → 解約まで継続利用

例:
- AI チャット使い放題（月 1,980 円）
- 限定コンテンツ閲覧（月 2,980 円）
- ツール利用（月 980 円）
```

メリット: 安定収入・複利で増える
デメリット: 解約されないように常に価値を提供する必要

#### ③ フリーミアム
```
基本機能は無料 → 高度な機能・利用回数増加で課金

例:
- 無料: 月 5 回まで
- 有料: 無制限（月 1,500 円）

例:
- 無料: 基本機能
- 有料: AI 機能・広告非表示（月 980 円）
```

メリット: ユーザー獲得が楽 → 一部が課金
デメリット: 「無料で使えるならいいや」と思われがち

#### ④ 投げ銭・寄付
```
ユーザーが任意で「これいいね」と思ったら払う

例:
- BOOTH の投げ銭
- note のサポート機能
```

メリット: 心理的ハードル低い
デメリット: 収益が安定しない

### あなたの場合、どれが合う?

#### 自己紹介・ポートフォリオサイト
→ 直接的な収益化は難しい。**「お問い合わせ」から仕事受注** が現実的

#### サービス LP
→ **月額課金（サブスク）** がベスト

#### ブログ・コンテンツ
→ **フリーミアム** か **投げ銭**

#### 教材・素材販売
→ **一括払い**

迷ったら、**月額 980 円のフリーミアム** から始めるのが王道です。

## レッスン 7-2: Stripe アカウントを作る（10 分）

### Stripe とは

世界最大の **オンライン決済サービス**。
- 個人でも法人でも使える
- アカウント開設無料
- 売上の **3.6%** が手数料（業界最安級）
- 日本円も対応
- 不正検知・領収書発行も自動

### アカウント開設手順

1. [https://stripe.com/jp](https://stripe.com/jp) を開く
2. 「今すぐ始める」をクリック
3. メール・名前・パスワードで登録
4. **メール認証**

→ これで **テストモード** が使えるようになります。
本番モードへの切り替えは「事業者情報」を入力した後。

### テストモードでまずは練習

最初は **テストモード** で動作確認します（実際にはお金が動かない）。
本番化は Week 7 の最後 or Week 8 の頭で OK。

### Stripe ダッシュボードで取得するキー

1. ログイン後、右上の **「テストモード」** が ON になっていることを確認
2. 左メニュー「Developers → API keys」
3. 以下の 2 つをコピー:
   - **Publishable key**（`pk_test_xxx`）
   - **Secret key**（`sk_test_xxx`）← 「Reveal」を押す

`.env.local` に追加:
```
STRIPE_PUBLISHABLE_KEY=pk_test_xxx
STRIPE_SECRET_KEY=sk_test_xxx
```

## レッスン 7-5: 値段の決め方（10 分）

### 「いくらにすればいい?」の答え

#### 安すぎる罠

```
❌ 月 100 円にしよう（誰でも払えるから...）
   ↓
   ・運営コストを賄えない
   ・「安物」と認識されて価値が伝わらない
   ・サポートに時間を取られて疲弊
```

#### 高すぎる罠

```
❌ 月 1 万円にしよう（プロっぽいから...）
   ↓
   ・誰も払わない
   ・最初の 1 人を獲得するハードルが高すぎる
```

### 適正価格の考え方

#### 公式
```
適正価格 = ユーザーが得られる価値 ÷ 10
```

例:
- ユーザーが月 5 万円分の時間を節約 → **月 5,000 円** が適正
- ユーザーが月 1 万円稼げるツール → **月 1,000 円** が適正

#### 個人開発の鉄板価格帯

| プラン | 価格目安 | 例 |
|------|---------|---|
| ライト | 月 500〜980 円 | 個人趣味用ツール |
| スタンダード | 月 1,500〜3,000 円 | 副業ツール |
| プロ | 月 5,000〜10,000 円 | 本格ビジネスツール |
| 教材買い切り | 5,000〜30,000 円 | ノウハウ・素材 |

### 最初は「強気の価格設定」が正解

最初に安く設定すると、後から値上げしにくいです。
**最初から「適正価格」**で始めましょう。

「高いかな?」と思う価格は、実は **多くの人にとって妥当**。

## レッスン 7-6: テスト決済で全体を確認（15 分）

### テストモードでの決済

Stripe テストモードでは、以下のテストカード番号が使えます:

```
番号: 4242 4242 4242 4242（成功）
有効期限: 任意の未来の日付（例: 12/35）
CVC: 任意の 3 桁（例: 123）
名前: 任意

→ 必ず決済成功する
```

その他テストカード:
```
4000 0000 0000 0002 → 必ず決済失敗（一般エラー）
4000 0000 0000 9995 → 残高不足エラー
4100 0000 0000 0019 → 不正利用検出
```

### 全体フローを通してみる

```
1. サイトを開く（公開された URL）
2. 「プレミアム会員になる」ボタンをクリック
3. Stripe Checkout 画面が表示される
4. テストカード番号 4242 4242 4242 4242 を入力
5. 決済完了 → サンキューページにリダイレクト
6. マイページに戻ると「プレミアム会員」と表示
7. 有料コンテンツが見られる ✨
8. Stripe Dashboard の「Payments」に決済が記録されている
```

これが通れば、**本番で本当のカードを使っても同じように動きます**。

## Stripe アカウント作成手順書

### Step 1: アカウント作成
1. https://stripe.com/jp を開く
2. 「今すぐ始める」をクリック
3. メール・パスワードで登録
4. メール認証

### Step 2: ダッシュボード探検
- 左メニュー: Payments / Customers / Products / Subscriptions / Developers
- 上部右: テストモード ⇔ 本番モード切替

### Step 3: API キー取得
1. Developers → API keys
2. テストモードで「Reveal test key」をクリック
3. Publishable key と Secret key をコピー
4. `.env.local` に貼り付け

### Step 4: 商品 / プラン作成（サブスクの場合）
1. Products → Add product
2. 名前・説明・価格・課金頻度を入力
3. 作成された Price ID をメモ
4. Claude に渡してサブスク実装

### Step 5: Webhook 設定
1. Developers → Webhooks → Add endpoint
2. URL: `https://your-app.vercel.app/api/stripe/webhook`
3. Listen to events: `checkout.session.completed`, `customer.subscription.*`, `invoice.payment_failed`
4. 作成された Signing secret をコピー
5. `.env.local` に `STRIPE_WEBHOOK_SECRET=whsec_xxx` を追加

### Step 6: 本番化（テストが OK だったら）
1. アカウント設定で事業者情報を入力
2. 銀行口座を登録
3. 本人確認書類アップロード
4. 数日で承認 → 本番モードに切替

## トラブルシューティング

### Q1. テスト決済で「Webhook signature verification failed」と出る

**A**: `STRIPE_WEBHOOK_SECRET` が未設定 or 間違っている。
- Stripe Dashboard → Webhooks → 該当エンドポイント → Signing secret を再コピー
- `.env.local` と Vercel に貼り直し

### Q2. 決済は成功するが、ユーザーのプランが更新されない

**A**: Webhook が動いていない。
1. Stripe Dashboard → Webhooks の「Last delivery」を確認
2. 失敗していたら、Claude にエラー文を渡して修正依頼
3. ローカル開発時は Stripe CLI で `stripe listen --forward-to localhost:3000/api/stripe/webhook` を使う

### Q3. 本番モードに切り替えたら決済が動かない

**A**:
- 本番用 API キーに切り替えたか確認
- `STRIPE_PUBLISHABLE_KEY` が `pk_live_xxx` で始まっているか
- 商品・プランも本番モードで作り直したか
- Webhook も本番用 URL で再設定したか

### Q4. 解約フローが面倒くさい

**A**: Stripe Customer Portal を使う:
```
Stripe Customer Portal を実装して、
ユーザーが自分でカード変更・解約・請求書ダウンロードできるようにして
```

### Q5. 売上の入金がいつ来るか分からない

**A**: Stripe デフォルト設定では:
- 売上発生 → 7 日後に決済確定
- そこから週 1 回まとめて銀行口座に入金（毎週月曜が多い）

「Settings → Payouts」で頻度・最低金額を変更可能。

### Q6. 領収書はどう発行する?

**A**: Stripe が自動で発行・メール送信します。
顧客が「Customer Portal」から PDF ダウンロードも可能。
