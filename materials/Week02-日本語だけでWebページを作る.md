# Week 2: 日本語だけで Web ページを作る

> **このWeekのゴール**: 自分だけの Web サイト（複数ページ・自分で内容を編集できる）が、ブラウザに表示される
> **所要時間**: 動画 50 分 + 教材 90 分 + 宿題 30 分 = 約 2 時間 50 分
> **このWeekで身につく力**: 「Web サイトという完成品を、自分の手で生み出す力」

---

## このWeek を始める前に

### 振り返り: Week 1 で何ができるようになったか

- ✅ Claude Code が PC で動いている
- ✅ 日本語で頼んだら、簡単なアプリが作れた
- ✅ 「上手に頼むコツ」を 5 つ覚えた

### Week 2 で取り組む変化

```
Week 1: 1 つの画面で動くアプリ（メモ帳・電卓 等）
       ↓
Week 2: 複数のページを持つ Web サイト（自己紹介・サービス紹介 等）
       ↓
       これが「Web サービスの土台」になります
```

Web サイトと Web アプリの違いを 1 行で:

> **Web サイト** = 見るだけ（パンフレットのような）
> **Web アプリ** = 操作できる・データが保存される（道具のような）

Week 2 ではまず **「Web サイト」** をしっかり作れるようになり、Week 4 で **「Web アプリ」** に進化させます。

## レッスン 2-1: 何を作るか決めよう（15 分）

### 「自分のための Web サイト」を 1 つ選んでください

8 週間かけて育てていく **「あなたの作品」** を、ここで決めます。
完璧でなくて大丈夫。途中で変えても大丈夫。**まず 1 つ選ぶこと** が大事です。

#### A. 自己紹介サイト（おすすめ・初心者最適）

```
- 自分のプロフィール
- 経歴・スキル
- 過去の実績
- お問い合わせフォーム
```

**メリット**: 内容を考えやすい。すぐ作れる。卒業後も使える（ポートフォリオサイトとして）

#### B. サービス LP（収益化したい方向け）

```
- サービスの紹介
- 料金プラン
- お客様の声
- お申し込みボタン
```

**メリット**: Week 7 の決済まで一直線。副業・起業を考えている方に最適。

#### C. 個人ブログ・note 風サイト

```
- 記事一覧
- 記事詳細ページ
- カテゴリ分類
- 著者プロフィール
```

**メリット**: 書くのが好きな方に。SEO で集客もできる。

#### D. 趣味のポートフォリオ

```
- 作品ギャラリー
- 作品詳細
- 自己紹介
- 連絡先
```

**メリット**: イラスト・写真・音楽など趣味のある方に。

### 決め方のコツ

> **迷ったら、A の自己紹介サイトを選んでください**。
> 一番作りやすく、Web の基本がしっかり身につきます。

## レッスン 2-2: 「Web サイトを作って」と頼む（20 分）

### 作業フォルダの準備

ターミナルで以下を実行:

```bash
cd ~/my-apps
mkdir my-website
cd my-website
claude
```

**意味**:
1. `cd ~/my-apps` → my-apps フォルダに移動
2. `mkdir my-website` → my-website というフォルダを新規作成
3. `cd my-website` → そのフォルダに入る
4. `claude` → Claude Code を起動

### サンプル: カフェの紹介サイト（実演用）

まずは練習として、Claude に以下のように頼んでみてください:

```
おしゃれなカフェの紹介サイトを作って。

ページ構成:
- トップページ（カフェの雰囲気写真・店名・コンセプト）
- メニュー紹介
- 店舗情報（住所・営業時間・地図）
- お問い合わせフォーム

デザイン:
- 落ち着いた茶色系の配色
- 写真を大きく使う
- 高級感のある見た目
- スマホでも綺麗に見える
```

→ Claude が 1〜2 分で作ってくれます。

### 完成したらブラウザで開く

Claude が「`index.html` を開いてください」と言ったら:

1. `my-website` フォルダを開く（Mac は Finder、Windows はエクスプローラ）
2. `index.html` をダブルクリック
3. ブラウザで開く

→ **本物のカフェのサイトが表示される** ✨

### このサンプルで実感してほしいこと

たった 1 つの指示で、**4 ページの Web サイトが、洗練されたデザインで** 出来上がります。
これが Claude Code の威力です。

## レッスン 2-3: ブラウザで確認する方法（10 分）

### 方法 ①: ファイルをダブルクリック（一番簡単）

`index.html` をダブルクリック → ブラウザが開く。
これで完了です。

### 方法 ②: Claude に「開いて」と頼む

```
作ったサイトをブラウザで開いて
```

→ Claude が自動でブラウザを開いてくれます。

### 方法 ③: ローカルサーバーで開く（少し本格的）

これは Week 3 以降で必要になることがあります。今は知識として頭の片隅に。

```
このサイトをローカルサーバーで起動して
```

→ Claude が `http://localhost:3000` のような URL を出してくれます。これをブラウザに貼り付けると見られます。

### 「変更が反映されない！」と思ったら

**ブラウザのキャッシュ** が原因です。以下のショートカットで強制再読み込み:

- **Mac**: `⌘ + Shift + R`
- **Windows**: `Ctrl + Shift + R`

これで最新の状態が表示されます。

## レッスン 2-4: 内容を変更する頼み方（20 分）

カフェのサイトはあくまで練習。**ここから自分のサイトに育てていきます**。

### 自己紹介サイトに作り変える例

```
このサイトを、私の自己紹介サイトに変えて。

私の情報:
- 名前: 山田太郎
- 職業: 元営業マン → AI 個人開発者を目指す
- 趣味: カフェ巡り、競馬予想
- 一言: 「AI で人生を変えたい 40 代」

ページ構成は:
- トップ（プロフィール写真・名前・キャッチコピー）
- 自己紹介（経歴・スキル）
- 作ったもの（後で追加していく予定）
- お問い合わせ

配色は紺×ゴールドで、落ち着いた高級感のある感じで。
```

→ サイトの内容が一気に書き変わります。

### 部分的に変更する頼み方

```
✅ 「店名（カフェ・ド・パリ）を、山田太郎の個人サイトに変えて」
✅ 「ヒーロー画像を、自分のプロフィール写真用のスペースに変えて」
✅ 「メニュー紹介ページを、『作ったもの』ページに変えて」
✅ 「営業時間のところを削除して」
✅ 「お問い合わせフォームの項目を、名前・メール・お問い合わせ内容の 3 つに減らして」
```

**コツ**: 「全体的に変えて」よりも「ここを → こうして」の方が確実です。

### 文章の生成も AI に任せる

```
「自己紹介の文章を、温かみのある感じで 3 段落書いて。私は元営業マンで、これから AI 開発を学んで個人事業を始めたい 45 歳。」
```

→ Claude が自然な日本語で文章を書いてくれます。気に入らなければ「もっとカジュアルに」「もっとユーモアを入れて」など修正。

## レッスン 2-5: ページを増やす（15 分）

### 新しいページを追加する

```
「ブログ」というページを追加して。
記事一覧が表示されて、各記事のタイトル・日付・抜粋が見られるようにして。
最初は記事 3 件のサンプルでいいです。
```

→ ヘッダーのナビゲーションに「ブログ」が追加され、記事一覧ページが新規作成されます。

### よく追加するページ

| ページ名 | 頼み方 |
|---------|--------|
| ブログ | 「ブログページを追加して。記事一覧 → 記事詳細の構成で」 |
| 料金プラン | 「料金プランページを追加して。3 つのプラン（ベーシック・スタンダード・プレミアム）で」 |
| よくある質問 | 「FAQ ページを追加して。アコーディオン形式で開閉できる質問 5 つ」 |
| お客様の声 | 「お客様の声ページを追加して。3 名の体験談を写真付きで」 |
| 会社概要 | 「会社概要ページを追加して。会社情報・代表メッセージ・沿革を含めて」 |
| プライバシーポリシー | 「プライバシーポリシーページを追加して」 |
| 特定商取引法 | 「特定商取引法に基づく表記ページを追加して」 |

### ナビゲーションメニューを整える

```
ヘッダーのナビゲーションを、
「トップ」「自己紹介」「作ったもの」「お問い合わせ」の 4 つに整理して。
スマホでは、右上の三本線（ハンバーガーメニュー）を押すと開くようにして。
```

→ プロっぽいヘッダーが完成します。

## コラム: 「内容」と「見た目」のどちらが先か

これはよく聞かれる質問です。

**答え: 内容が先、見た目は後**。

なぜなら、見た目から決めると「内容に合わない」「結局やり直し」になりがち。
逆に、内容（構成・文章）が固まっていれば、デザインは Week 3 で AI に頼むだけで何パターンも作れます。

Week 2 では **「自分のサイトに何を書くか・どんな構成にするか」** に集中してください。
カッコいいデザインは Week 3 で作ります。
