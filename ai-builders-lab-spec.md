# AI Builders Lab - 詳細仕様書

> **ブランド方針**: サービス名は **AI Builders Lab**。講師の実績は一般名詞「**競馬予想AI**」として語る。固有のサービス名「Dlogic」は LP・公開ドキュメント上では使用しない。

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| サービス名 | AI Builders Lab |
| コンセプト | ノーコードで本格競馬予想AIを作った実績者が教える、AI個人開発者養成コミュニティ |
| メインメッセージ | 「10年前には不可能だった。今日から、あなたにもできる。」 |
| サブメッセージ | 「ノーコードで本格競馬予想AIを作った私が、その全てを教えます。題材は競馬AIですが、身につくのは『何でも作れる力』です。」 |
| ターゲット | 競馬好き×AI興味層 / AI副業・起業を目指す層 |

## 6. TOPページ仕様（/）

### セクション構成

#### Section 1: ヒーロー
- メインキャッチ：「10年前には不可能だった。今日から、あなたにもできる。」
- サブキャッチ：「ノーコードで本格競馬予想AIを作った私が、その全てを教えます。題材は競馬AIですが、身につくのは『何でも作れる力』です。」
- CTA ボタン：「無料ローンチ動画を見る」→ `/launch/episode/1` へ
- 背景：黒ベース＋ゴールドのグラデーション or パーティクルエフェクト

#### Section 2: 時代の変化（共感・問題提起）
- 見出し例：「AIは、個人開発の常識を変えた」
- コンテンツ：
  - コードを書かなくてもプロ品質のアプリが作れる時代
  - まだほとんどの人がその事実を知らない
  - 気づいた人から、圧倒的な差がつき始めている
- ビジュアル：タイムライン or 対比（10年前 vs 今）

#### Section 3: 実績・証拠
- 見出し例：「私がノーコードで作ったもの」
- コンテンツ：
  - 講師の代表作「本格競馬予想AI」をフィーチャー枠で大きく扱う（固有のサービス名は出さない）
  - その他開発実績（ガチャゲーム・AIカウンセラー・データ分析・コンテンツ自動生成・スワイプ型LP SaaS・EC買取等）
  - 「全てClaude Codeだけで開発」という事実
- ビジュアル：フィーチャーカード+その他カード一覧

#### Section 4: AI Builders Labとは
- 見出し例：「AI Builders Labは、作れる人間を増やす場所です」
- コンテンツ：
  - 競馬予想AIを題材にしながら教えるが、本質は「何でも作れる力」を身につけること
  - 学ぶだけでなく、仲間と作り、依頼し合えるコミュニティ
  - 3つのキーワード：学ぶ・作る・稼ぐ
- ビジュアル：アイコン＋テキストの3カラム

#### Section 5: カリキュラム概要
- 見出し例：「8週間で、あなたは変わる」
- コンテンツ：8週間のタイムライン形式
  - Week 1：Claude Codeとは・環境構築・AIへの指示の出し方
  - Week 2：サービス設計の考え方・仕様書の書き方
  - Week 3：Next.js＋Supabaseの基本構成
  - Week 4：フロントエンド制作（LP・UI）
  - Week 5：予測ロジック・スコアリング設計
  - Week 6：決済・サブスク機能の実装
  - Week 7：デプロイ・本番運用・VPS管理
  - Week 8：集客・マネタイズ・次のプロダクトへの展開
- ビジュアル：縦型タイムライン or ステップカード

#### Section 6: 料金プラン
- 見出し例：「あなたのペースで選べる3つのプラン」
- コンテンツ：3プランカード（動画のみ・動画＋メール・Zoom）
  - 各プランの内容・価格を明記
  - 推奨プランをゴールドでハイライト
- CTA：「今すぐ申し込む」→ `/apply` へ

#### Section 7: よくある質問（FAQ）
- アコーディオン形式
- 想定Q例：
  - プログラミング経験がなくても大丈夫ですか？
  - 競馬の知識がなくても受講できますか？
  - 動画はいつまで視聴できますか？
  - Zoom型は録画を後から見られますか？
  - 返金保証はありますか？

#### Section 8: 申し込みCTA
- 見出し例：「さあ、作れる側の人間になろう」
- サブテキスト：期生募集人数・締め切り日を記載（動的に変更可能）
- CTAボタン：「申し込む」→ `/apply` へ
- フッター：運営者情報・特定商取引法・プライバシーポリシー

## 7. ローンチページ仕様（/launch）

### トップ（/launch）
- 4話のサムネイル一覧
- 話数順に並べる
- 視聴済みバッジ（クッキーまたはローカルストレージで管理）
- 第4話のみ第3話視聴後にアンロック（任意）

### 各話ページ（/launch/episode/[n]）

| 話 | タイトル | 内容 | 目的 |
|---|---|---|---|
| 第1話 | 「AIが変えた、個人開発の常識」 | 時代の変化・AIで何でも作れる現実 | 共感・興味喚起 |
| 第2話 | 「競馬予想AIはこうして生まれた」 | 講師の開発実録・使ったツール・失敗談 | 信頼構築・実績提示 |
| 第3話 | 「あなたにも同じことができる理由」 | 受講者目線・学習ロードマップ・卒業後のビジョン | 欲求喚起・不安解消 |
| 第4話 | 「AI Builders Lab、始まります」 | サロン全貌・料金・申し込み方法 | クロージング・CTA |

### 各話ページのレイアウト
- 動画プレイヤー（Cloudflare Stream埋め込み）
- 話のタイトル・説明テキスト
- 前の話・次の話へのナビゲーション
- 第4話のみ：申し込みボタンを動画下に設置
- メールアドレス登録フォーム（任意）→ Supabaseに保存

## 8. 申し込みページ仕様（/apply）

- プラン選択（3プラン）
- 選択後Stripeの決済画面へ遷移
- 決済完了後：
  - Supabaseにユーザー作成
  - 確認メール送信（Supabase Auth）
  - `/members/dashboard` へリダイレクト

## 9. 会員エリア仕様（/members）

### 認証・アクセス制御
- Supabase Auth（メール＋パスワード）
- プランごとにアクセス可能ページを制御
  - 動画のみ：動画ライブラリ・資料・マイページ
  - メール付き：上記＋質問フォーム
  - Zoom型：全ページ＋Zoom予約リンク

### 9-1. ダッシュボード（/members/dashboard）
- ウェルカムメッセージ（名前表示）
- 受講進捗バー（何週目まで完了か）
- お知らせ一覧（管理者が投稿）
- 次回Zoomセミナーの日程・参加リンク（Zoom型のみ）
- クイックリンク（動画・資料・コミュニティ）

### 9-2. 動画ライブラリ（/members/videos）
- 週ごとにグループ化（Week 1〜8）
- 週ごとのアンロック制御（管理者が手動またはスケジュールで解放）
- Cloudflare Stream埋め込みプレイヤー
- 視聴済みチェックマーク
- 動画ごとの説明テキスト・補足メモ

### 9-3. 資料ダウンロード（/members/materials）
- カテゴリ別（仕様書テンプレート・プロンプト集・参考資料）
- ファイル形式：PDF・MD・ZIP
- Supabase Storageからダウンロード
- プランによるアクセス制御

### 9-4. コミュニティ（/members/community）
- 掲示板形式（スレッド投稿・返信）
- カテゴリ：自己紹介・質問・作ったもの報告・雑談
- いいね機能
- 管理者による投稿削除・ピン留め機能

### 9-5. 質問・サポート（/members/support）
- メール相談フォーム（動画＋メール以上のプランのみ）
- 送信内容：件名・本文・ファイル添付（任意）
- Supabaseにチケット保存
- 管理者がメールで返信（外部メーラー使用）
- 過去の相談履歴一覧

### 9-6. 受発注ボード（/members/jobs）
- 投稿形式：依頼したい / 受けたい の2種類
- 投稿内容：タイトル・概要・予算・期間・連絡方法
- 検索・フィルター（カテゴリ・予算・新着順）
- 興味あり / 応募ボタン → 投稿者に通知
- 管理者（仁さん）への直接依頼ボタン

### 9-7. マイページ（/members/mypage）
- プロフィール編集（名前・自己紹介・SNSリンク）
- 現在のプラン確認
- 請求履歴（Stripe連携）
- プランアップグレード
- 退会申請

## 12. Supabaseテーブル設計（概要）

### usersテーブル
```

```

```
id, week, title, description, cloudflare_video_id, unlocked_at, created_at
```

```
id, title, category, file_url, plan_required, created_at
```

### postsテーブル（コミュニティ）
```
id, user_id, category, title, body, pinned, created_at
```

### repliesテーブル
```
id, post_id, user_id, body, created_at
```

### ticketsテーブル（サポート）
```
id, user_id, subject, body, status, created_at
```

### jobsテーブル（受発注）
```
id, user_id, type(依頼/受注), title, description, budget, duration, contact, created_at
```

### announcementsテーブル
```
id, title, body, created_at
```
